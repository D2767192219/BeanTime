# 拼豆管理平台 - 规格文档

## 1. 概述

- **项目名称**: 拼豆管理平台
- **项目类型**: 单 HTML 文件离线应用
- **核心功能**: 店铺桌台计时、预约、结算管理
- **目标用户**: 拼豆店店员

## 5. 核心功能

### 5.1 桌台管理
- 区域用字母编号（A=大厅，B=单人间，C=包间...），字母按创建顺序固定
- 桌台编号格式为字母+数字（如 A1、B2）
- 每个区域内的桌台按数字从小到大排序
- 主内容区按区域字母顺序分组展示，每组有标题（A 大厅 4桌）
- 批量建台（起始编号+数量+区域）
- 支持新增/删除区域（管理区域弹窗）
- 支持编辑桌台（可修改名称、区域、标签）
- 桌台管理按钮（新建桌台、删除桌台）保持统一大小

### 5.2 业务流程（固定状态流转，不可手动选择状态）
```
空闲 ─[立即开台]→ 选豆中 ─[开始计时]→ 使用中
  │                      │
  │                      └─[暂停]→ 暂停中 ─[继续]→ 使用中
  └──[客户预约]→ 已预约 ─────────────────┘
                                    │
                              [结束]→ 结算 → 回到空闲
```
- **立即开台**：空闲→选豆中，生成6位随机会话ID
- **客户预约**：空闲→已预约，生成6位随机会话ID，同时开始计时
- **开始计时**：选豆中/已预约→使用中，开始或继续计时
- **暂停/继续**：使用中↔暂停中
- **结束**：使用中/暂停中→结算→回到空闲

### 5.3 会话ID
- 每次「立即开台」或「客户预约」时自动生成6位随机ID（如 `K7X2PQ`）
- 会话ID显示在卡片座位号下方
- 结算后会话ID清除，桌台恢复空闲
- 结算记录中保留会话ID用于追溯

### 5.4 预约时间显示
- 客户预约后开始计时，显示预约开始时间
- 点击"开始计时"按钮后，预约计时清零重新开始计时
- 预约状态显示"预约时间"，其他计时状态显示"开始时间"

### 5.5 更换桌台
- 可与任意状态的台桌互换（不只是空闲台桌）
- 两个台桌完全互换状态、计时信息、会话ID
- 更换界面显示目标台桌的标签和状态信息

### 5.6 搜索功能
- 顶部 header 集成搜索栏
- 支持按桌台名称、标签或会话ID实时筛选
- 搜索结果仍按区域分组展示

### 5.7 筛选导航
- 左侧栏显示各状态分类及数量
- 点击筛选显示对应状态桌台
- 标签筛选支持按标签快速筛选
- 主内容区按区域字母顺序分组

### 5.8 设置面板
- 计时费率设置（元/小时）
- 弹窗编辑

### 5.9 营业额面板
- 今日营业额
- 累计营业额
- 结算记录包含会话ID

## 6. 技术栈

- **HTML5** 单文件结构
- **CSS3** 内嵌样式，无外部依赖
- **Vanilla JavaScript** ES6+
- **IndexedDB** 本地持久化存储
- **无任何外部 CDN / 依赖**

## 7. 交互规范

- 响应式网格布局，主内容区横向4列排列
- 所有操作即时保存到 IndexedDB
- 计时器每秒更新显示
- 弹窗使用原生 dialog 元素
- 状态变化时即时更新侧边栏和状态条计数
